# Culturalbility Financial Model

## Revenue Goal

**Target:** ¥100,000 per month (~$14,000 USD)

## Path to 100K

### Assumptions
- Average Revenue Per User (ARPU): ¥50/month
- Target subscribers needed: **2,000 paying users**
- Freemium conversion rate: 3-5%
- Free users needed: 40,000-67,000 MAU

### Timeline

| Phase | Months | Milestone | Revenue |
|-------|--------|-----------|---------|
| **Development** | 1-3 | Build MVP | ¥0 |
| **Beta** | 4 | 100 beta testers | ¥0 |
| **Launch** | 5-6 | App Store release | ¥3,000/mo |
| **Growth** | 7-12 | Marketing push | ¥30,000/mo |
| **Scale** | 13-18 | Enterprise + partnerships | ¥100,000/mo |

**Total time to goal: 18-24 months**

## Cost Structure (Monthly)

| Category | Cost (¥) | Notes |
|----------|----------|-------|
| AI/LLM API | 5,000-15,000 | Scales with users |
| Cloud hosting | 2,000-5,000 | AWS/Aliyun |
| App store fees | 2,500 | Apple 30%, Google 15% |
| Marketing | 10,000-30,000 | Ads, influencers |
| **Total costs** | **20,000-50,000** | |

**Profit margin at 100K: 50-80%**

## Key Metrics to Track

- CAC (Customer Acquisition Cost): Target < ¥25
- LTV (Lifetime Value): Target > ¥150
- Churn rate: Target < 5%/month
- Daily Active Users / MAU ratio: Target > 20%

## Risk Factors

1. **AI costs could eat margins** — need efficient architecture
2. **Competition** — Duolingo, other AI language apps
3. **Regulation** — China content/audio AI regulations
4. **Payment friction** — WeChat/Alipay integration complexity

## Next Steps

1. Validate pricing with 50 user interviews
2. Calculate precise COGS (Cost of Goods Sold) per user
3. Define "premium" features worth paying for
4. Plan viral/growth mechanics before building
